L'unité centrale: accès à la mémoire principale

M1 1 bit

Ils vous permettent de sélectionner la valeur à utiliser pour mettre à jour le registre de données (MBR): avec la valeur de la mémoire ou la valeur du bus de données interne.

MH 1 bit

Ce signal permet de sélectionner le compteur de performances matérielles (HPC) avec lequel travailler:

  • 00: Nombre de cycles d'horloge.
  • 01: Temps CPU cumulé.
  • 10: Consommation électrique cumulée.
  • 11: <Non utilisé>.

Le Hardware Performance Counter est un ensemble de registres spéciaux pour stocker des statistiques sur les activités liées au matériel. Il est utilisé pour effectuer des analyses de performance ou de consommation d'énergie de bas niveau.

R 1 bit
W 1 bit
BE 4 bits
MRdy 1 bit

Signaux de commande de la mémoire principale: R en lecture, W en écriture, BE en sélection d'octets.
Par défaut, la mémoire lit ou écrit en un cycle d'horloge. Si la mémoire est configurée pour plusieurs cycles d'horloge, le signal MRdy est défini lorsque la mémoire met fin à la demande R/W.

L'unité centrale: Fichier de registre (RF).

REMARQUE: Ce mécanisme est uniquement destiné à des tests. Vous ne devez pas utiliser ces signals dans la version finale de votre microcode.
Veuillez éviter d'utiliser directement RA, RB, RC et Cop, car sa valeur est générée par SelA (SelB, SelC, SelCop) et MR.

RA 5 bits

RB 5 bits

RC 5 bits

LC 1 bit

Ils vous permettent de sélectionner un registre dont la valeur passe à A (RA) ou B (RB) ou de mettre à jour la valeur de l'enregistrement sélectionné (RC). Dans le cas de la mise à jour du registre, en plus d'utiliser le signal RC, utilisez le signal LC avec la valeur 1.

L'unité centrale: Registres

C0, ..., C7 1 bit

Il va stocker dans le registre associé, la valeur de l'entrée du registre à la fin du cycle d'horloge.

The central processing unit: Access to the data bus

Td 1 bit

Copy the value of the MBR register, after being transformed by the 'bytes selector' componente, into the data bus.

The central processing unit: Access to the address bus

Ta 1 bit

Copy the value from the MAR register into the address bus.

The central processing unit: Access to the internal data bus

T1, ..., T12 1 bit

Copy the value in the input of the associated tri-state element into the internal data bus, during the clock cycle.

The central processing unit: The ALU flags, plus I and U flags

NOTE: this mechanism is for testing purposes only. You should not use those signals in the final version of your microcode.
Please avoid to use TEST_C, ... directly because its value are generated by the ALU.

TEST_C, ..., TEST_U 1 bit

Let user to change on-the-fly the values of flags as they were generated by the hardware elements.
The current available flags are:

  • C: Carry
    • Add a b: (a >>> 31) && (b >>> 31)
  • V: oVerflow
    • Add a b: (a+b < 0) && (a >= 0) && (b >= 0) || (a+b >= 0) && (a < 0) && (b < 0)
  • N: Negative
    • Add a b: (a+b < 0)
  • Z: Zero
    • Add a b: (a+b == 0)

  • I: Interruption enabled/Interruption disabled
  • U: User level/Kernel level

ExCode 4 bits

Let microprogrammer to specify a inmediate value that should be copy into the internal data bus with signal T11.

The Input/Output Elements

IOR 1 bit
IOW 1 bit
IORdy 1 bit

Control signals for the I/O devices: IOR for read, IOW for write, and if the devices is asynchronous the IORdy signal is set when device ends the request.
